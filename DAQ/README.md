# DAQ Folder

This folder contains the data acquisition and control scripts for the indentation setup:

- `force_acq.py`: reads force data from the SPI-connected ADC on the Raspberry Pi and writes a live feed to `/tmp/force_feed.txt`
- `motor_control1.py`: controls the Z motor through `pyximc` and writes displacement data to `/tmp/motor_feed.txt`
- `gui.py`: desktop GUI that combines force, motor, plotting, export, optional camera, and optional XY stage bridge control
- `run_indent_cli1.py`: command-line indentation workflow for automated approach, detect, retract, slow indent, CSV export, and plotting
- `mqtt_publisher.py`: tails the motor/force feed files and publishes them over MQTT
- `kim101_client.py`: HTTP client for a separate KIM101 bridge service
- `common.py`: shared constants

## What This Folder Needs

This code is intended to run on a Raspberry Pi connected to the DAQ hardware.

Hardware assumptions in the current code:

- SPI ADC on `/dev/spidev0.0`
- GPIO data-ready pin on BCM `17`
- GPIO trip input on BCM `22`
- GPIO soft-limit input on BCM `26`
- XIMC-controlled motor reachable through a URI such as `xi-com:///dev/ttyACM0`

The GUI can also talk to an XY stage bridge over HTTP using `--kim http://host:port`.

## Python Requirements

Core modules used here:

- `numpy`
- `matplotlib`
- `spidev`
- `RPi.GPIO`
- `paho-mqtt`
- `requests`
- `Pillow`
- `opencv-python` or system `cv2` package
- `tkinter`

The motor code also expects the `pyximc` Python wrapper and matching XIMC libraries to be available. `motor_control1.py` already tries to find `pyximc` relative to the current repo layout.

Example install on Raspberry Pi:

```bash
python3 -m pip install numpy matplotlib spidev paho-mqtt requests pillow opencv-python
sudo apt install python3-tk
```

`RPi.GPIO` is usually installed already on Raspberry Pi OS. `pyximc` installation depends on the local XIMC SDK setup.

## Important Output Files

These scripts communicate through live text feed files:

- `/tmp/force_feed.txt`
- `/tmp/motor_feed.txt`

`force_acq.py` writes rows in this form:

```text
time_s counts volts mN
```

`motor_control1.py` writes motor position data to:

```text
time_s displacement_mm ...
```

Other scripts in this folder read from those files while the system is running.

## Recommended Ways To Run

### 1. Run the GUI

Main GUI command:

```bash
python3 gui.py --uri xi-com:///dev/ttyACM0
```

If you also use the KIM101 bridge:

```bash
python3 gui.py --uri xi-com:///dev/ttyACM0 --kim http://192.168.5.3:5005
```

What the GUI does:

- starts the force worker
- starts the motor worker
- reads live force and motor feeds
- provides plots and export tools
- can optionally publish data and use a camera if those dependencies are installed

### 2. Run the automated CLI indentation workflow

```bash
python3 run_indent_cli1.py --uri xi-com:///dev/ttyACM0
```

Useful example with explicit settings:

```bash
python3 run_indent_cli1.py \
  --uri xi-com:///dev/ttyACM0 \
  --speed_fast_mm_s 0.10 \
  --approach_budget_mm 1.0 \
  --detect_threshold_mN 0.10 \
  --retract_after_detect_um 20 \
  --speed_slow_mm_s 0.010 \
  --indent_slow_mm 0.30 \
  --max_force_mN 0.50
```

This script:

- starts force acquisition
- connects to the motor
- approaches the sample
- detects surface contact from force threshold
- retracts slightly
- performs a slower indentation
- saves a CSV
- opens a force-vs-displacement plot

### 3. Run only the MQTT publisher

Use this only after force and motor feeds are already being generated by the GUI or CLI workflow.

```bash
python3 mqtt_publisher.py
```

Common environment variables:

```bash
export MQTT_HOST=localhost
export MQTT_PORT=1883
export MQTT_TOPIC=MON
export DEVICE_ID=your_device_id
export DEVICE_TOKEN=your_device_token
export FORCE_PUBLISH_UNIT=mn
python3 mqtt_publisher.py
```

### 4. Quick standalone motor worker test

```bash
python3 motor_control1.py
```

This is only a simple worker/connectivity test, not the normal user workflow.

## Typical Workflow

1. Power the DAQ electronics, ADC, and motor controller.
2. Confirm the motor device is present, for example `/dev/ttyACM0`.
3. Run the GUI or the CLI workflow from this folder.
4. Check that `/tmp/force_feed.txt` and `/tmp/motor_feed.txt` are updating.
5. If needed, start `mqtt_publisher.py` in another terminal.


## Notes For Other Users

- Run all commands from inside the `DAQ` folder.
- These scripts are hardware-dependent and are not expected to work on a non-Raspberry Pi machine without modification.
- Calibration values for force conversion are defined directly in `force_acq.py`.
- Feed files in `/tmp` are overwritten during new runs, so copy exported results elsewhere if they need to be kept.
